5  The Model DSL

5.1 Overview

ferx models live in plain-text .ferx files. Each file has up to seven sections in a fixed order. Three sections are required; the rest have sensible defaults.

Section Required Purpose
[parameters] Yes Declare thetas, omegas, sigmas with bounds
[individual_parameters] Yes Express individual PK params in terms of thetas + ETAs
[structural_model] Yes Analytical PK shorthand or ODE declaration
[odes] If ODE model Differential equations (one per state)
[error_model] Yes Residual error distribution
[initial_values] No Override starting values without editing the bound lines
[fit_options] No Estimation method, max iterations, covariance

5.2 [parameters]

Thetas use the form theta NAME(typical, lower, upper). The three numbers are: typical value (starting estimate), lower bound, upper bound.

[parameters]
  theta TVCL(0.134, 0.001, 10.0)   # CL typical value, bounded (0.001, 10)
  theta TVV(8.1,   0.1,   500.0)
  theta TVKA(1.0,  0.01,  50.0)

Omegas declare between-subject variability:

  omega ETA_CL ~ 0.07    # variance (not SD) — initial estimate
  omega ETA_V  ~ 0.02
  omega ETA_KA ~ 0.40

The ~ syntax means “variance of this ETA”. ferx always treats the omega matrix as diagonal unless you use the block syntax (see Section 14.1).

Sigma declares residual variance:

  sigma PROP_ERR ~ 0.01   # proportional error variance

The optional (sd) qualifier lets you specify the initial value on the standard-deviation scale instead of variance — usually more intuitive for sigmas and IIV alike:

  sigma PROP_ERR ~ 0.10 (sd)     # 10% on the SD scale -> var = 0.01
  omega  ETA_CL  ~ 0.30 (sd)     # 30% CV on the log scale

The estimate, the init_as_sd column in ferx_estimates(), and the printed output all preserve this distinction so users always know which scale the initial value was declared on.

5.2.1 Block / correlated random effects

For correlated ETAs (or kappas), use the block form with the column-major lower-triangle of the covariance matrix:

[parameters]
  block_omega (ETA_CL, ETA_V) = [0.09, 0.02, 0.04]   # var(CL), cov(CL,V), var(V)
  block_kappa (KAPPA_CL, KAPPA_V) = [0.02, 0.005, 0.01]

The three values for a 2×2 block are [var1, cov12, var2] — column-major lower triangle (subscripts: row, col). For a 3×3 block the six values are [var1, cov21, cov31, var2, cov32, var3], where cov_ij means row=i, col=j.

Use a block omega when CL and V are expected to co-vary (e.g. allometric subjects with both higher CL and larger V). A positive off-diagonal cov12 means subjects with higher-than-typical CL also tend to have higher-than-typical V. The diagonal-only default (omega ETA_CL ~ 0.09) assumes independence.

ferx_cor_matrix(fit) reports the estimated correlation matrix for a block omega after fitting. The bundled warfarin_block_omega example demonstrates this; see ferx_example("warfarin_block_omega").

5.2.2 kappa — inter-occasion variability

The kappa keyword declares an occasion-level random effect; see Section 14.1.

5.2.3 Fixing parameters (FIX)

Append FIX to any theta, omega, sigma, or kappa declaration to hold the parameter at its initial value during estimation:

[parameters]
  theta TVCL(0.134, 0.001, 10.0)
  theta TVLAG(0.25, 0.0, 2.0) FIX   # lag time fixed at 0.25
  omega ETA_CL ~ 0.07
  omega ETA_KA ~ 0.40 FIX            # BSV on KA fixed
  sigma PROP_ERR ~ 0.01 (sd) FIX     # sigma fixed on SD scale
  kappa KAPPA_CL ~ 0.05 FIX          # IOV variance fixed

Placement is flexible. FIX may appear at the end of the line (after the closing parenthesis) or, for theta, inside the parentheses:

theta TVLAG(0.25, 0.0, 2.0) FIX   # end of line — traditional NONMEM style
theta TVLAG(0.25, FIX)             # two-element form: init + FIX, no bounds
theta CL (FIX) (0.1, 0.001, 10.0) # FIX anywhere before the bound triple
omega ETA_CL (FIX) ~ 0.07         # FIX before the tilde

Fixed parameters appear in print(fit) and ferx_estimates() with SE = FIXED and %RSE = FIXED; they are excluded from the covariance matrix.

Theta bounds. The full form is (initial, lower, upper). You may omit the upper bound when there is no natural ceiling:

theta CL(0.134, 0.001, 10.0)   # three-element: init, lower, upper
theta CL(0.134, 0.001)         # lower-bound only; no upper bound

Unused-parameter warning. Declaring a parameter in [parameters] but not referencing it in [individual_parameters] or [error_model] triggers a warning-severity message with category "unused_parameter". This usually indicates a commented-out expression or a typo. Inspect ferx_warnings(fit) or run ferx_model_validate() before fitting.

5.3 [individual_parameters]

Each line assigns an individual PK parameter using thetas and ETAs.

[individual_parameters]
  CL = TVCL * exp(ETA_CL)   # log-normal BSV
  V  = TVV  * exp(ETA_V)
  KA = TVKA * exp(ETA_KA)

Any expression valid in R-like syntax is accepted: arithmetic, exp(), log(), inv_logit(), logit(), covariate columns from the dataset.

5.3.1 Covariate effects

Use dataset column names directly in expressions:

[individual_parameters]
  CL = TVCL * (WT / 70)^THETA_WT * (CRCL / 100)^THETA_CRCL * exp(ETA_CL)
  V1 = TVV1 * (WT / 70)^THETA_WT * exp(ETA_V1)

WT and CRCL are read from the dataset row for each subject. The power-model form (cov / ref)^theta is the conventional allometric scaling.

5.3.2 Conditional logic (if / else)

[individual_parameters] supports if/else for covariate-driven branching.

Block form — use when the body contains multiple assignments or more than two branches:

[individual_parameters]
  if (WT > 70) {
    CL = TVCL * (WT / 70)^0.75 * exp(ETA_CL)
  } else {
    CL = TVCL * exp(ETA_CL)
  }
  V = TVV * exp(ETA_V)

Inline (ternary) form — use for a single-value branch:

[individual_parameters]
  CL = if (SEX == 1) TVCL * 1.5 * exp(ETA_CL) else TVCL * exp(ETA_CL)
  V  = TVV * exp(ETA_V)

Conditions support <, <=, >, >=, ==, !=, &&, ||, !. Both forms can be combined freely in the same block.

Note: ferx skips mu-reference detection for parameters assigned inside an if block, since the ETA→THETA relationship is not unconditional there. Unconditional assignments in the same block are unaffected.

The bundled warfarin_if example demonstrates a two-branch allometric CL model; see ferx_example("warfarin_if").

5.4 [structural_model]

5.4.1 Analytical PK shortcuts

ferx provides built-in analytical solutions for common compartment models:

Keyword Cmts Route Parameters
one_cpt_oral(cl, v, ka) 1 Oral cl, v, ka
one_cpt_iv_bolus(cl, v) 1 IV bolus cl, v
one_cpt_infusion(cl, v) 1 IV infusion cl, v
two_cpt_oral(cl, v1, q, v2, ka) 2 Oral cl, v1, q, v2, ka
two_cpt_iv_bolus(cl, v1, q, v2) 2 IV bolus cl, v1, q, v2
two_cpt_infusion(cl, v1, q, v2) 2 IV infusion cl, v1, q, v2
three_cpt_oral(cl, v1, q2, v2, q3, v3, ka) 3 Oral cl, v1, q2, v2, q3, v3, ka
three_cpt_iv_bolus(cl, v1, q2, v2, q3, v3) 3 IV bolus cl, v1, q2, v2, q3, v3
three_cpt_infusion(cl, v1, q2, v2, q3, v3) 3 IV infusion cl, v1, q2, v2, q3, v3

All analytical models accept an optional lagtime= (or alag=) argument; oral models additionally accept an optional f= bioavailability argument. See Section 15.7 and Section 15.9.

These are invoked with named arguments:

[structural_model]
  pk one_cpt_oral(cl=CL, v=V, ka=KA)

The pk keyword tells the parser to use the analytical solution engine. See the ferx-core PK models reference for derivations.

5.4.2 ODE models

When no analytical solution exists, declare ode(...) and write ODEs in [odes]:

[structural_model]
  ode(obs_cmt=central, states=[depot, central])

[odes]
  d/dt(depot)   = -KA * depot
  d/dt(central) = KA * depot / V - VMAX * central / (KM + central)

obs_cmt names the compartment whose amount is used for the DV prediction. See Section 11.1 for a full ODE example.

5.5 [error_model] 

[error_model]
  DV ~ proportional(PROP_ERR)

Supported error distributions:

Syntax Model When to use
proportional(sigma) DV = IPRED * (1 + eps) Concentration data, relative variability
additive(sigma) DV = IPRED + eps Log-scale or count data
combined(sigma_prop, sigma_add) Both Transitional concentration ranges
log_additive(sigma) Additive on log scale; DV already log-transformed Importing pre-logged data

eps is drawn from N(0, sigma) for each observation. All sigma parameters are on the standard-deviation scale.

For log-transform-both-sides (LTBS) with DV on the natural scale, use the alternative declaration log(DV) ~ additive(sigma) — the engine log-transforms the DV column at load time. Use DV ~ log_additive(sigma) when the dataset already contains log-transformed values. See Section 15.10 for full details.

For multi-endpoint models (simultaneous PK/PD), prefix each line with CMT=N: to assign a distinct error model per compartment:

[error_model]
  CMT=2: DV ~ proportional(PROP_ERR_PK)
  CMT=3: DV ~ additive(ADD_ERR_PD)

See Section 15.10 for LTBS details and Section 12.1 for per-CMT error models.

5.6 [initial_values]

The [initial_values] section lets you override starting estimates without touching the bound lines. This is convenient for re-running after a first fit:

[initial_values]
  theta = [0.133, 7.89, 1.49]
  omega = [0.0425, 0.0162, 0.328]
  sigma = [0.0162]

Values must be listed in the same order the parameters appear in [parameters].

5.7 [fit_options] 

[fit_options]
  method     = focei    # foce | focei | saem | gn | gn_hybrid
  maxiter    = 300
  covariance = true

Omit this section to use the defaults (method = foce, maxiter = 300, covariance = false).

5.8 Optional blocks

A complete reference of advanced blocks is in Section 15.1; the short form:

Block Purpose Chapter
[scaling] Unit conversion (obs_scale) and per-CMT readouts (y[CMT=N] = ...) Section 15.1
[diffusion] Stochastic differential equations (process noise) Section 15.11
[covariate_nn NAME] Neural-network covariate effect (Deep Compartment Models) — (advanced)
[simulation] Embedded simulation trial design Section 8.1

5.9 Validating a model

Before running a long fit, check syntax:

library(ferx)
ferx_model_validate(ferx_example("warfarin")$model)
Validating: warfarin.ferx 

Sections present:
  parameters                     [ok]
  individual_parameters          [ok]
  structural_model               [ok]
  error_model                    [ok]
  fit_options                    [ok] (optional)

Result: VALID

If a section is missing or malformed, the validator reports which section failed and Result: INVALID. The return value is TRUE / FALSE, so the function can be used programmatically.
