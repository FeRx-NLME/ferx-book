2  Installation

2.1 Prerequisites

ferx requires R ≥ 4.2, Rust (stable toolchain), and cmake. Enzyme (LLVM) is optional — it enables exact automatic differentiation for faster gradients, but is not needed to get started.

Dependency Required Notes
R ≥ 4.2 Yes Package interface
Rust / Cargo Yes Compile the ferx-core backend; stable toolchain is sufficient
cmake Yes Used by the Rust build (Linux/macOS; bundled with Rtools45 on Windows)
Enzyme (LLVM) No Faster gradients via autodiff; documented in Section 2.5

Two installation paths are described below:

  • Quick install (Section 2.2) — stable Rust + finite-difference gradients. Works on all platforms. Identical estimates to the autodiff path, slightly slower. Start here.
  • Full Enzyme install (Section 2.5) — a custom Rust fork with the Enzyme autodiff plugin. Recommended once you start fitting larger models or using AD-only features.

2.1.1 Two separate one-time costs

The “full install takes hours” reputation conflates two distinct, one-time steps. Knowing which you are paying for avoids confusion:

Step What it is Roughly When
1. Build the Enzyme toolchain Compile a custom nightly rustc + LLVM with Enzyme from source (Section 2.5) ~45–60 min Once per machine. The toolchain is reused forever (lives at ~/.local/share/enzyme-toolchain). Autodiff path only.
2. Compile ferx Build the ferx / ferx-core Rust crates and link them highly CPU-dependent (see below) First install only. Cargo caches dependencies, so later reinstalls/updates take minutes.

Step 2’s duration depends almost entirely on single-core CPU speed, because the autodiff build’s final link is a single-threaded fat-LTO pass (extra cores do not help). As a rough guide:

  • Fast modern CPU (recent desktop, or native Apple Silicon arm64): ~15–30 min.
  • Older / mobile laptop CPU (e.g. a 2019 Intel i7, which also thermal-throttles under sustained single-core load): up to ~1–2 h.

The finite-difference build is much quicker either way — it skips fat LTO and uses thin LTO. So:

  • First-time autodiff user pays both steps → roughly 1.5–3 h end-to-end, dominated by your CPU.
  • First-time finite-difference user pays only a faster version of step 2 (no step 1; thin LTO links noticeably faster).
  • Returning user pays neither — both are cached.
Tip

If you would rather not compile at all, the Docker image (Section 2.6) ships a prebuilt toolchain and package — no local Rust/Enzyme build, no wait.

Step 2 is slower on the autodiff path specifically because of that fat LTO, which is required for cross-crate Enzyme correctness — see the note in Section 2.5.

2.2 Quick install by platform

2.2.1 macOS

cmake is the only non-standard system dependency. A stable Rust toolchain is sufficient.

Step 1 — Install Homebrew (skip if already installed)

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Step 2 — Install cmake

brew install cmake

Step 3 — Install Rust

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

Follow the on-screen instructions to add Rust to your PATH (usually source "$HOME/.cargo/env", or restart the terminal).

Step 4 — Install ferx from R

RStudio on macOS launches with a restricted PATH that does not include the Homebrew cmake bin directory, even when the shell profile is configured correctly. Add it explicitly before installing:

# Apple Silicon (M1/M2/M3):
Sys.setenv(PATH = paste("/opt/homebrew/opt/cmake/bin", Sys.getenv("PATH"), sep = ":"))
# Intel Mac — use this line instead:
# Sys.setenv(PATH = paste("/usr/local/opt/cmake/bin", Sys.getenv("PATH"), sep = ":"))

Sys.setenv(FERX_NO_AUTODIFF = "1")
pak::pak("FeRx-NLME/ferx-r")
# or: devtools::install_github("FeRx-NLME/ferx-r")

Without the Sys.setenv(PATH = ...) line, the Rust build fails with a cmake not found error when launched from RStudio.

2.2.2 Windows

Native Windows installs are supported with finite-difference gradients. The ferx-r build auto-detects the missing Enzyme toolchain and falls back automatically — no FERX_NO_AUTODIFF=1 flag is needed on Windows. All estimation methods work; AD-only features are unavailable.

Native Enzyme autodiff is not available on Windows because the custom Rust fork does not currently build there. For autodiff on Windows, use the Docker image (Section 2.6), WSL2 + Ubuntu, or a remote Linux host.

Step 1 — Install Rtools45

Rtools45 ships cmake and the MinGW gcc that R uses to link the package, so no separate cmake install is needed.

Step 2 — Install rustup with the GNU-ABI toolchain

Download from https://www.rust-lang.org/tools/install, then in PowerShell:

rustup toolchain install stable-x86_64-pc-windows-gnu

You do not need to rustup default it — the package’s build pins this toolchain automatically on Windows. (The rustup default on Windows is the MSVC ABI, which is not link-compatible with Rtools’ MinGW linker.)

Step 3 — Install ferx from R

pak::pak("FeRx-NLME/ferx-r")
# or: devtools::install_github("FeRx-NLME/ferx-r")

First install compiles the Rust backend; how long depends heavily on your CPU (~15–30 min on a fast modern machine, up to ~1–2 h on an older laptop — see Section 2.1.1). Subsequent installs are fast.

2.2.3 Linux

Install cmake and Rust via your system package manager plus rustup. RStudio on Linux inherits the system PATH, so no PATH workaround is needed.

# Debian / Ubuntu
sudo apt-get install -y cmake build-essential curl git
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
Sys.setenv(FERX_NO_AUTODIFF = "1")
pak::pak("FeRx-NLME/ferx-r")
# or: devtools::install_github("FeRx-NLME/ferx-r")

2.3 Verify the installation

library(ferx)
ferx_example()          # lists all bundled examples
ex <- ferx_example("warfarin")
fit <- ferx_fit(ex$model, ex$data)
print(fit)

The first install compiles the Rust backend; the time is CPU-dependent (~15–30 min on a fast machine, up to ~1–2 h on an older laptop — see Section 2.1.1). print(fit) leads with a STATUS: CONVERGED line and bold section headers; ferx_warnings(fit) shows any warnings grouped by severity with remediation guidance. Subsequent installs reuse Cargo’s artifact cache and are much faster.

2.4 The FERX_NO_AUTODIFF flag

Setting FERX_NO_AUTODIFF = "1" skips the Enzyme autodiff toolchain probe and forces finite-difference gradients. FD is slightly slower but produces identical estimates. LLVM/Enzyme are not needed. All examples in this book run with FERX_NO_AUTODIFF=1 on CI.

The flag is only required when you want to skip the probe in scripts or CI; the build will automatically fall back to FD when no Enzyme toolchain is found.

Two additional flags control the autodiff preflight self-test (see Section 2.5):

  • FERX_AD_PREFLIGHT_SKIP=1 — skip the AD self-test (useful when reinstalling on a toolchain you have already verified; the result is also cached per toolchain fingerprint so the test is skipped automatically on matching reinstalls).
  • FERX_AD_PREFLIGHT_TIMEOUT=<seconds> — timeout budget for the preflight compile-and-run (default: 90 s).

2.5 Advanced: full Enzyme install

Enzyme unlocks exact gradients via automatic differentiation of the Rust source, and is required for a few AD-only features (e.g. SAEM HMC proposals via n_leapfrog). The toolchain is a custom fork of nightly Rust built from source.

  • Total time: ~45–60 min toolchain build + a CPU-dependent first ferx-r compile (~15–30 min on a fast machine, up to ~1–2 h on an older laptop — see Section 2.1.1).
  • One-time setup. Once steps 1–4 are done, the Enzyme toolchain lives at ~/.local/share/enzyme-toolchain and survives reboots and /tmp purges. To reinstall or update ferx-r later, you only need step 5.
NoteThe first autodiff compile is slow — and the end looks like a hang

The autodiff build uses fat LTO (lto = "fat"), which is required for cross-crate Enzyme correctness: the ferx-core functions being differentiated are inlined into the final ferx crate so Enzyme can see through the call boundary. Fat LTO re-optimizes the whole dependency graph in one pass, so the final crate (Compiling ferx …) routinely takes many minutes with no further per-crate progress output — this is normal, not a stall. Confirm it is working with ps aux | grep rustc (you’ll see one busy rustc --crate-name ferx process).

This is distinct from the quick install path (Section 2.2), which uses thin LTO and links noticeably faster. Only the first autodiff compile pays the full cost; Cargo caches dependencies, so later reinstalls are fast.

If rustc instead pins a core at 100% CPU during the autodiff verification step (Section 2.5, Step 3) and never returns, that is a different problem — an LLVM/Enzyme mismatch — covered there.

  • macOS
  • Linux
  • Windows

2.5.0.1 Step 1. Build dependencies

xcode-select --install                       # clang, git (skip if already installed)
brew install cmake ninja python3             # needed by the rust build system

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"
rustup toolchain install nightly --profile minimal

2.5.0.2 Step 2. Build the Enzyme-enabled Rust toolchain from source (~45–60 min)

git clone --depth 1 https://github.com/rust-lang/rust /tmp/rust-src
cd /tmp/rust-src

# IMPORTANT: --set llvm.download-ci-llvm=false avoids the LLVM/Enzyme mismatch
# that causes rustc to hang silently when compiling autodiff code.
./configure \
    --release-channel=nightly \
    --enable-llvm-enzyme \
    --enable-llvm-link-shared \
    --enable-ninja \
    --disable-docs \
    --set llvm.download-ci-llvm=false

./x build --stage 1 library                  # the long step (~45–60 min)

# Register the freshly built toolchain with rustup under the name "enzyme"
rustup toolchain link enzyme build/host/stage1
rustc +enzyme --version --verbose            # sanity check: prints version info

2.5.0.3 Step 3. Verify autodiff works (fat LTO is mandatory for Enzyme)

cat > /tmp/ad_check.rs <<'EOF'
#![feature(autodiff)]
use std::autodiff::autodiff_forward;

#[autodiff_forward(df, Dual, Const, Dual)]
#[inline(never)]
fn f(x: f64, y: f64) -> f64 { (x * y).exp() + x * x }

fn main() {
    let (val, dfdx) = df(1.5, 1.0, 2.0);
    let expect = 2.0 * (1.5 * 2.0_f64).exp() + 2.0 * 1.5;
    assert!((dfdx - expect).abs() < 1e-9);
    println!("autodiff OK: f={val:.4}, df/dx={dfdx:.4}");
}
EOF

rustc +enzyme -Zautodiff=Enable -Clto=fat -Copt-level=2 /tmp/ad_check.rs -o /tmp/ad_check
/tmp/ad_check

Expected: compiles in seconds, prints autodiff OK: f=22.3355, df/dx=43.1711.

Warning

If rustc hangs at low/constant memory for many minutes (RSS stays under ~100 MB and never grows), the LLVM/Enzyme link is broken. Rebuild Step 2, making sure --set llvm.download-ci-llvm=false is set.

2.5.0.4 Step 4. Move the toolchain to a permanent location (macOS purges /tmp)

mkdir -p ~/.local/share

# -L dereferences the LLVM/Enzyme dylib symlinks into real files, so the copy
# still works after /tmp/rust-src is deleted. Cycle warnings are harmless.
cp -aL /tmp/rust-src/build/host/stage1 ~/.local/share/enzyme-toolchain

# Prune the bundled rust source tree (only used for IDE source browsing).
# Without this the copy is ~14-30 GB; after pruning it is ~640 MB.
rm -rf ~/.local/share/enzyme-toolchain/lib/rustlib/rustc-src \
       ~/.local/share/enzyme-toolchain/lib/rustlib/src

du -sh ~/.local/share/enzyme-toolchain        # should be ~640 MB

# Re-point the rustup "enzyme" toolchain at the permanent path
rustup toolchain uninstall enzyme
rustup toolchain link enzyme ~/.local/share/enzyme-toolchain

# Re-verify against the new path
rustc +enzyme -Zautodiff=Enable -Clto=fat -Copt-level=2 /tmp/ad_check.rs -o /tmp/ad_check && /tmp/ad_check

# Reclaim ~8 GB now that the permanent copy is verified
rm -rf /tmp/rust-src

2.5.0.5 Step 5. Install ferx-r (in R; first compile is CPU-dependent, ~15 min–2 h; fast thereafter)

pak::pak("FeRx-NLME/ferx-r")
# or: devtools::install_github("FeRx-NLME/ferx-r")
Note

Before the long fat-LTO compile, the package automatically runs the same #[autodiff_forward] compile-and-run self-test from Step 3, under a timeout watchdog. A broken toolchain aborts in seconds with a clear message rather than wedging rustc deep into the build. Set FERX_AD_PREFLIGHT_SKIP=1 to skip this on reinstalls of a toolchain you have already verified (the result is also cached per toolchain fingerprint — see Section 2.4).

Tip

As long as /tmp/rust-src is gone but ~/.local/share/enzyme-toolchain survives, you only ever need to repeat Step 5 to reinstall or update ferx-r.

Tested on Ubuntu 22.04 / 24.04. Other distributions work with package-manager adjustments (substitute apt for dnf, pacman, etc.).

sudo snap remove rustup 2>/dev/null || true

Same as the macOS Step 3 above — compile and run /tmp/ad_check.rs with rustc +enzyme -Zautodiff=Enable -Clto=fat -Copt-level=2.

2.5.0.9 Step 4. Move the toolchain to a permanent location

/tmp is cleared on reboot on most Linux distros (and by systemd-tmpfiles periodic cleanup on Ubuntu), which would silently break the toolchain link.

mkdir -p ~/.local/share
cp -aL /tmp/rust-src/build/host/stage1 ~/.local/share/enzyme-toolchain
rm -rf ~/.local/share/enzyme-toolchain/lib/rustlib/rustc-src \
       ~/.local/share/enzyme-toolchain/lib/rustlib/src
du -sh ~/.local/share/enzyme-toolchain        # should be ~640 MB

rustup toolchain uninstall enzyme
rustup toolchain link enzyme ~/.local/share/enzyme-toolchain

rm -rf /tmp/rust-src

For multi-user / shared-host installs at /opt/rust-nightly, see INSTALL-SYSADMIN.md in the ferx-r repository.

2.5.0.10 Step 5. Install ferx-r

pak::pak("FeRx-NLME/ferx-r")
Note

Before the long fat-LTO compile, the package automatically runs the same #[autodiff_forward] compile-and-run self-test from Step 3, under a timeout watchdog. A broken toolchain aborts in seconds with a clear message rather than wedging rustc deep into the build. Set FERX_AD_PREFLIGHT_SKIP=1 to skip this on reinstalls of a toolchain you have already verified (the result is also cached per toolchain fingerprint — see Section 2.4).

The Enzyme-enabled Rust fork does not currently build on native Windows. Use one of:

  • WSL2 + Ubuntu — wsl --install -d Ubuntu, then follow the Linux workflow inside the Ubuntu shell.
  • Docker image (Section 2.6) — bundles a pre-built Enzyme toolchain.
  • Remote Linux host — SSH into a Linux server or cloud VM.

If Enzyme is available the install log will contain:

[build] autodiff: Enzyme backend enabled

2.6 Docker / reproducible environments

The ferx-r repo ships a Dockerfile that bundles the Enzyme toolchain (built from source), the ferx CLI, the ferx R package, and RStudio Server — no local Rust/Enzyme setup required. On Windows this is the recommended path if you need Enzyme autodiff.

git clone https://github.com/FeRx-NLME/ferx-r
cd ferx-r

# Build (first build takes ~45–60 min; cached after that)
docker build -t ferx:latest .

# Run RStudio Server
docker run --rm -p 8787:8787 -e PASSWORD=ferx ferx:latest
# -> open http://localhost:8787   user: rstudio   password: ferx

For CI without Enzyme, a minimal image is sufficient:

FROM rocker/r-ver:4.4.0
ENV FERX_NO_AUTODIFF=1
RUN Rscript -e "pak::pak('FeRx-NLME/ferx-r')"
