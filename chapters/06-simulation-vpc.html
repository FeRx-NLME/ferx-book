7  Simulation & VPC

7.1 Overview

ferx_simulate() generates replicated datasets from the fitted model. The primary use case is the Visual Predictive Check (VPC): overlay observed data on simulation-based prediction intervals to assess model adequacy visually.

library(ferx)
library(ggplot2)
library(dplyr)

ex  <- ferx_example("warfarin")
obs <- read.csv(ex$data)
fit <- ferx_fit(ex$model, ex$data, method = "focei", covariance = TRUE)
Warning: Model file [fit_options] sets `method = foce` but ferx_fit() argument
overrides it with `focei`. The call-time value will be used.
Mu-referencing detected for: ETA_CL, ETA_KA, ETA_V

7.2 Simulating from the fitted model

The typical VPC workflow passes fit to ferx_simulate() so that simulation uses the fitted theta, omega, and sigma rather than the model file’s initial values:

sim <- ferx_simulate(
  model  = ex$model,
  data   = ex$data,
  n_sim  = 500,
  seed   = 42,
  fit    = fit
)
head(sim)

n_sim = 500 replicates is sufficient for a standard VPC. Set seed for reproducibility.

7.3 What the simulation returns

Column Meaning
SIM Replicate index (1 to n_sim)
ID Subject ID (same population structure as observed data)
TIME Time (matched to observation schedule in data)
IPRED Individual prediction for this replicate (no residual error)
DV_SIM Simulated observation (IPRED + residual noise)

7.4 Simulating at initial values (without a fit)

Omit fit to simulate at the initial estimates in the model file. Useful for a prior predictive check:

sim_prior <- ferx_simulate(ex$model, ex$data, n_sim = 200, seed = 1)

7.5.2 Manual ggplot2 VPC

For full control over appearance, compute the prediction intervals yourself:

# Summarise simulation by time bin
time_breaks <- c(0, 1, 2, 4, 6, 8, 12, 24, 36, 48, 72, 96, 120)

sim_binned <- sim |>
  mutate(time_bin = cut(TIME, breaks = time_breaks, labels = FALSE, right = FALSE)) |>
  filter(!is.na(time_bin)) |>
  group_by(time_bin, SIM) |>
  summarise(
    p05 = quantile(DV_SIM, 0.05, na.rm = TRUE),
    p50 = quantile(DV_SIM, 0.50, na.rm = TRUE),
    p95 = quantile(DV_SIM, 0.95, na.rm = TRUE),
    TIME = mean(TIME),
    .groups = "drop"
  ) |>
  group_by(time_bin) |>
  summarise(
    lo  = median(p05),
    mid = median(p50),
    hi  = median(p95),
    TIME = mean(TIME),
    .groups = "drop"
  )

obs_dv <- obs |> filter(EVID == 0, !is.na(DV)) |> mutate(DV = as.numeric(DV))

ggplot() +
  geom_ribbon(data = sim_binned, aes(x = TIME, ymin = lo, ymax = hi),
              fill = "#1F6B7A", alpha = 0.25) +
  geom_line(data = sim_binned, aes(x = TIME, y = mid),
            colour = "#1F6B7A", linewidth = 1) +
  geom_point(data = obs_dv, aes(x = TIME, y = DV),
             alpha = 0.5, size = 1.5) +
  scale_x_continuous(breaks = c(0, 24, 48, 72, 96, 120)) +
  labs(x = "Time (h)", y = "Concentration (mg/L)",
       title = "VPC — warfarin 1-cpt oral") +
  theme_minimal()

7.7 Interpreting the VPC

A good VPC shows:

  • Observed data scattered within the shaded band
  • Roughly 10% of points outside the 5th–95th percentile band
  • The observed median tracking the simulated median line
  • No systematic trends over time or concentration ranges

If the observed percentiles consistently fall outside the simulated band, the model is mis-specifying the distribution (consider a different error model or structural model).
